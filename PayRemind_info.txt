При установке:
PaymentBaseHelper.USING_COPY_DB = 1;
после компиляции проекта -
поддерживается извлечение прилагаемой БД 
(из каталога .../app/src/main/assets)
т.е. здесь есть файл БД с уже подготовленными данными
(первый запуск - стартует с "exception", затем - всё
работает корректно с заполненной новой БД из
проекного каталога "assets").

Если же: PaymentBaseHelper.USING_COPY_DB = 0; // Normal mode
Все работает как обычно - приложение создаёт
(при первом запуске) Базу Данных.
Таким образом - вносим изменения в БД.

После этого - в Android Studio (через Device File Explorer) -
копируем файл: "paymentBase.db" на наш диск и
далее - переносим в наш проект, в каталог "assets". 

Посему, если БД готова - то первые два запуска с:
PaymentBaseHelper.USING_COPY_DB = 1;
После чего - установить данную переменную в 0 и
пересобрать проект!

30.04.2023 Полный уход от старых пакетов java.util.Date и
java.util.Calendar. Теперь все - сделано при помощи:
java.time.LocalDate и java.time.LocalDateTime.
Также в таблицу БД "payments" - добавлено новое поле:
"execdate" - дата выполнения последнего платежа.
В данном поле храниться дата и время выполнения
последней оплаты по выбранному платежу. Если же последнего
платежа не было (или была очистка дат платежей) - 
там будет 0 (начало отсчёта: 1970-01-01). При этом - если
дата за 1970 год, на дисплее отображается пустая строка.
При хранении даты в поле "execdate" -
формат этого поля тот же, что и для "date" - счетчик миллисекунд.
В кодах проекта - поле "date" имеет тип: java.time.LocalDate,
а поле "execdate" - java.time.LocalDateTime.
